В основе проекта лежат три класса-сущности:
* Пользователь (класс User);
* Тариф (класс Tariff);
* СМС-сообщение (класс Message);

Через определённый интервал времени происхоит уменьшение стоимости минуты разговора на величину, заданную в поле changePeriod тарифа.

API:

/tariffApp/users
/tariffApp/users/{id}
/tariffApp/tariffs
/tariffApp/messages/{id}

Обновление тарифов происходит в фоновом режиме по cron-расписанию (задаётся в файле database.properties параметром cron.check.update).
В данном случае задано выполнение  обновления ежедневно в 3.15.

Данные приложения хранятся в СУБД PostgreSQL. Параметры подключения к базе задаются в конфигурационном файле database.properties.
